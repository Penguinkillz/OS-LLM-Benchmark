Foundational methodology

Measuring the Control Plane of LLMs

Standardizing metrics for open-source large language model serving. This document outlines the mathematical frameworks and architectural assumptions used to derive our benchmark rankings.

query_stats

Statistical Robustness: p95 & p99

In distributed LLM serving, "average latency" is a deceptive metric. Because token generation is an autoregressive process with highly variable sequence lengths, latency distributions often exhibit heavy tails.

p95 Latency

The threshold below which 95% of token requests are fulfilled. This represents the 'standard' high-load experience for the majority of users.

p99 Latency

The threshold for the slowest 1%. Essential for identifying 'outlier' requests caused by garbage collection, memory swaps, or network jitter.

// Latency Aggregation Formula
def calculate_percentile(latencies, percentile):
  sorted_data = sorted(latencies)
  index = ceil(percentile / 100 * len(sorted_data)) - 1
  return sorted_data[index]

We prioritize p99 as the primary metric for system reliability, as LLM applications often require tight timeouts for real-time interactivity.

balance

Jain Fairness Index

How fairly does the scheduler distribute GPU resources among competing requests?

J(x) = (Σxᵢ)² / (n · Σxᵢ²)
Fairness Coefficient

A score of 1.0 indicates absolute fairness. In LLM serving, a low Jain Index suggests that long-prompt requests are starving shorter requests, or vice versa.

  • » n = Number of concurrent users
  • » x = Throughput (tokens/sec)

The OS Paradigm

Traditional LLM benchmarks focus on model accuracy (MMLU, GSM8K). OS-LLM-Benchmark shifts the focus to The Inference Kernel. We treat the LLM engine (vLLM, TGI, TensorRT-LLM) as the Operating System.

Why Operating System Thinking Applies

Modern LLM serving mimics process scheduling in Linux. Requests are "tasks" that require predictable CPU (Compute) and Memory (VRAM) allocation. Our benchmark evaluates the efficiency of the "Kernel" in maximizing hardware utilization without compromising on quality of service (QoS).
